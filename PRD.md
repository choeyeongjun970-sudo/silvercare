# 제품 요구사항 정의서 (PRD) - 실버케어 파트너스

## 1. 프로젝트 개요
- **프로젝트명**: 실버케어 파트너스 서비스 소개 사이트 구축
- **목표**: 고령 인구와 그 가족들에게 신뢰감 있고 따뜻한 브랜드 이미지를 전달하며, 7가지 핵심 서비스를 효과적으로 소개하고 상담 신청을 유도하는 단일 페이지 웹사이트 제작.
- **주요 가치 (Slogan)**: "모든 어르신이 존엄하고 행복한 생애 마무리를 맞이할 수 있도록"

## 2. 타겟 사용자 (Target Audience)
1. **주요 사용자 (Primary)**: 70대 이상의 시니어 사용자 (본인의 생애 마무리 계획)
2. **보조 사용자 (Secondary)**: 40-50대의 자녀 세대 (부모님을 위한 케어 서비스 탐색)

## 3. 정보 아키텍처 (Information Architecture)
사이트는 원페이지(Single Page) 구조로 다음 섹션을 포함한다:
1. **헤더**: 로고, 메뉴 내비게이션 (회사소개, 서비스, 특장점, 후기, 팀, 문의하기)
2. **히어로 섹션 (Hero)**: 메인 비주얼, 핵심 슬로건, 상담 신청 버튼
   - **핵심 요구사항**: 자녀 세대가 부모님에게 드리고 싶은 '최고의 효도'와 '안심'이라는 정서적 가치를 전달하는 문구 필수 포함.
3. **회사 소개 (About)**: 설립정보, 대표자, 비전과 미션
4. **서비스 소개 (Services)**: 7가지 핵심 서비스를 아이콘+카드 형식으로 노출
   - 생애 설계 상담, 의료 케어, 일상 지원, 사회적 연결, 디지털 자서전, 법률/행정 지원, 장례 토탈 케어
5. **특장점 (Why Silvercare)**: 브랜드 차별화 포인트 3가지
6. **고객 후기 (Testimonials)**: 실제 이용 사례 (신뢰성 확보)
7. **팀 소개 (Team)**: 대표 및 전문가 팀원 프로필
8. **문의하기 (Contact)**: 연락처 정보 및 온라인 상담 신청 폼
9. **푸터 (Footer)**: 카피라이트, 약관 링크, SNS 정보 등

## 4. 디자인 및 사용자 경험 (UX/UI) 요구사항
- **Aesthetics**: 따뜻한 베이지(Warm Beige), 부드러운 블루(Soft Blue), 그린(Green) 계열을 활용한 정서적 안정감 제공.
- **Accessibility (고령자 배포)**:
  - **정서적 소구 (Emotional Resonating)**: "직접 다 해드리지 못해 미안한 마음을 대신 전합니다", "자식의 마음으로 끝까지 곁을 지키겠습니다" 등 자녀의 효심과 안심을 자극하는 고유의 단어와 문구를 첫 화면에 배치.
  - **폰트**: 가독성을 최우선으로 하여 기본 폰트 크기를 크게 설정 (최소 18px 이상 권장).
  - **대비**: 텍스트와 배경의 명도 대비를 확실히 하여 인지 저하 최소화.
  - **조작**: 버튼 및 클릭 요소의 크기를 크게 하여 조작 편의성 증대.
- **Interactions**: 부드러운 스크롤 애니메이션과 페이드인 효과로 고급스러운 느낌 연출.
- **Responsiveness**: 모바일 기기에 최적화된 반응형 레이아웃 필수.

## 5. 기술 사양 (Technical Specifications)
- **Front-end**: HTML5, CSS3, JavaScript (Vanilla JS)
- **Library/Icons**: 부드러운 애니메이션 구현을 위한 CSS 프레임워크 또는 애니메이션 속성 활용, Remix Icon 등의 웹 아이콘 팩 사용.
- **Image Strategy**: Placeholder 이미지를 사용하되, 고령 사용자에게 적합한 따뜻한 무드의 알맞은 Alt 텍스트 포함.

## 6. 기능적 요구사항 (Functional Requirements)
- **Form Validation**: 상담 신청 폼의 필수 입력값(이름, 연락처 등) 체크 및 전송 완료 피드백(JS Alert 등).
- **Navigation**: 상단 메뉴 클릭 시 해당 섹션으로 부드럽게 스크롤 이동 (Smooth Scroll).
- **Active state**: 스크롤 위치에 따라 메뉴 하이라이트 표시.

## 7. 비기능 요구사항 (Non-Functional Requirements)

### 7.1 보안 (Security)
- **개인정보 보호**: 상담 신청 시 수집되는 개인정보(이름, 연락처 등)는 암호화된 통신(HTTPS)을 통해 전달되어야 한다.
- **데이터 최소화**: 서비스 목적 달성에 필요한 최소한의 정보만 수집하며, 개인정보 처리방침을 명시하여 사용자 신뢰를 확보한다.
- **보안 코딩**: XSS(Cross-Site Scripting), CSRF(Cross-Site Request Forgery) 등 일반적인 웹 취약점에 대비한 소스코드 검증을 실시한다.
- **상담 폼 오남용 방지**: 봇(Bot)에 의한 스팸 접수를 방지하기 위해 간단한 입력 검증 또는 필요시 캡차(CAPTCHA) 적용을 고려한다.

### 7.2 성능 지표 (Performance Metrics)
- **로딩 속도 (Page Speed)**: 
  - 최초 의미 있는 페인트(FCP) 시간 1.5초 이내 목표.
  - 고령 사용자의 네트워크 환경을 고려하여 초기 페이지 로드 용량 최적화 (이미지 압축 및 지연 로딩 활용).
- **Core Web Vitals**:
  - LCP (Largest Contentful Paint): 2.5초 이내.
  - CLS (Cumulative Layout Shift): 0.1 이하 (콘텐츠 로드 시 화면 흔들림 방지).
- **최적화**: 이미지 형식 최적화(WebP 등) 및 정적 자산(CSS, JS)의 경량화(Minification) 적용.
- **안정성**: 동시 접속자 100명 이상의 환경에서도 페이지 반응 속도 300ms 이내 유지.

## 8. 주요 제약 사항
- 외부 라이브러리 사용 최소화 (Vanilla 환경 지향)
- 고령 사용자의 인지 능력에 반하는 지나치게 복잡한 인터랙션 배제
